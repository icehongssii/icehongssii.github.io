테스트용 데이터베이스를 호스트머신에 있는 걸 사용하지 않고 컨테이너로 사용하면 너무 편한다. 근데 유닛 테스트를 실행 할 때마다 매번 데이터를 넣어줄 수도 없고 이럴 때는 어떡하나? 

title: docker-entrypoint-initdb.d/ 에다가 스크립트를 사용한다!
- . In this directory, `.sh`, `.sql`, and `.sql.gz` scripts are executed when the container is started for the first time.

FROM mysql:latest

# mydb를 선언하고 
ENV MYSQL_DATABASE mydb 

copy ./scripts ./docker-entrypoint-initdb.d/

아래는 scripts/create.sql

# mydb 안에서 테이블을 생성한다
CREATE TABLE IF NOT EXISTS mydb.myothertable (
  column_name VARCHAR(255)
);

INSERT INTO mydb.myothertable (column_name) VALUES ('other_value');
