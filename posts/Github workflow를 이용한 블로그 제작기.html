def post_to_html():
    for file in os.listdir("./blogs"):
        if not file.endswith(".md"):
            continue        
        html = md2.markdown_path(POSTS_PATH+"/"+file, extras=["metadata", "highlightjs-lang", "spoiler", "tables", 'fenced-code-blocks', "admonitions"])        
        post_detail_template = ENV.get_template("post.html")                
        post_title = file.split(".md")[0] + '.html'
        output_path = os.path.join(os.getcwd()+EXPORT_PATH, post_title)

        with open(output_path, 'w', encoding='utf-8') as f:
            f.write(post_detail_template.render({"meta": html.metadata, "html": html, "static_dir":STATIC_PATH}))

프레임워크 없이 웹 애플리케이션을 만든다는 것은 건물을 지을 때 기본적인 철골구조나 규격 없이 처음부터 모든 것을 직접 설계하고 건축하는 것과 비슷합니다.

비유를 조금 더 확장해보자면:

  • 프레임워크가 있는 경우: 건물을 지을 때, 이미 만들어진 철골구조와 규격에 따라 작업합니다. 예를 들어, 벽을 세우거나 창문을 설치할 때 표준화된 치수와 재료를 사용할 수 있습니다. 이렇게 하면 건축 과정이 훨씬 더 빠르고 효율적이며, 결과물의 품질도 일정하게 유지됩니다. 이처럼, 웹 프레임워크는 개발자가 복잡한 작업을 쉽게 할 수 있도록 기본 구조와 규칙을 제공하여, 더 빠르고 안정적으로 웹 애플리케이션을 만들 수 있게 도와줍니다.

  • 프레임워크가 없는 경우: 반면, 프레임워크 없이 건물을 짓는다면, 철골부터 전기 배선까지 모든 것을 직접 설계하고 만들어야 합니다. 이 과정에서 발생할 수 있는 오류도 많고, 시간이 많이 소요될 뿐만 아니라 결과물의 품질도 보장하기 어렵습니다. 마찬가지로, 웹 프레임워크 없이 애플리케이션을 만든다면, HTTP 요청을 처리하거나 데이터베이스와 상호작용하는 등의 기본적인 기능을 모두 처음부터 직접 구현해야 합니다. 이는 많은 시간과 노력이 필요하며, 코드의 복잡성도 증가하게 됩니다.

결론:

프레임워크는 건축에서의 철골구조처럼, 웹 애플리케이션 개발에서의 기본 구조와 규격을 제공합니다. 이를 통해 개발자는 반복적인 작업을 줄이고, 더 빠르고 안정적으로 애플리케이션을 개발할 수 있습니다. 만약 이러한 프레임워크가 없다면, 개발자는 모든 것을 처음부터 구현해야 하므로, 개발 속도가 느려지고 오류 가능성이 높아지며 유지보수가 어려워질 수 있습니다.
