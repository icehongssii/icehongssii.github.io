python dockerfile 환경변수

마지막수정 2024-11-14 18:00:32 - 최초작성2024-11-14 18:00
태그 python docker
tl;dr

👯‍♂️ python dockerfile에서 가장 많이 쓰이는 두가지 변수 의미

FROM python:3.12.1-slim-bookworm

ENV PYTHONDONTWRITEBYTECODE 1
ENV PYTHONUNBUFFERED 1

WORKDIR /usr/src/app
COPY ./requirements.txt .
RUN pip install --upgrade pip & pip install -r requirements.txt

COPY . .

👯‍♂️ PYTHONDONTWRITEBYTECODE=1

파이썬은 인터프리터 언어. 실행하기전에 바이트코들 컴파일되는데 이렇게 바이트코드로 변환된걸 .pyc에 저장함. 그리고 파이썬 파일 실행 할 때 마다 계속해서 .pyc를 생성하는게 아니라 저장된 pyc를 재사용해 다시 컴파일 하지 않아 실행 속도가 향상된다.

근데 이렇게 되면 __pycache__ 디렉토리안에 컴파일된 파이썬 코드들이 생기게 됨. 따라서 개발 환경에서는 이러한 디렉토리 생성을 피하고자 바이트 코드를 적지 않게한다.

요약

.pyc 파일은 프로그램을 더 효율적으로 실행하기 위해 생성됩니다. 하지만, 개발 중에 굳이 필요하지 않을 경우 PYTHONDONTWRITEBYTECODE 환경 변수를 설정해 .pyc 파일 생성을 막을 수 있습니다.

👯‍♂️ UNBUFFERED=1

버퍼링이 되면 stdout(표준 출력)이나 stderr(표준 에러)가 바로 출력되지 않고, 특정 크기만큼 데이터가 쌓일 때까지 기다렸다가 한꺼번에 출력됩니다. 예를 들어, 파일로 로그를 남기거나 네트워크를 통해 데이터를 전송할 때, 버퍼링된 출력은 실시간으로 보이지 않을 수 있어 디버깅에 불편할 수 있습니다.

예시 상황:

  • 버퍼링됨: 프로그램이 중간중간 상태를 출력하는데, 실행 중에는 아무것도 보이지 않다가 프로그램이 끝나야 출력이 보일 수 있습니다.
  • 버퍼링되지 않음: 프로그램이 진행될 때마다 출력이 즉시 보이기 때문에, 실시간으로 상태를 확인할 수 있습니다.

그래서 PYTHONUNBUFFERED=1 설정을 하면, 출력이 바로바로 터미널에 표시되도록 해주어 디버깅하거나 로그를 실시간으로 모니터링할 때 유용합니다.
